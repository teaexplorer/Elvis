```markdown
# Тестовое задание - Backend

Разработка API для работы с достижениями

## Необходимые технологии
- Любая опенсорсная СУБД (предпочтительно PostgreSQL)
- Python 3
- Flask/FastAPI
- Базовые алгоритмы
- Умение читать и понимать задание
- Навыки написания документации

## Основная задача
- Реализовать на Python 3 сервер (FastAPI или Flask) для взаимодействия с базой данных
- Разместить код на GitHub/GitLab
- Отправить ссылку на репозиторий в ответном письме

## Структура базы данных

### 1. Таблица пользователей
Обязательные поля:
- `username` — имя пользователя
- `language` — выбранный язык (ru/en)

### 2. Таблица достижений
Обязательные поля:
- `name` — название достижения
- `points` — количество баллов (целое положительное число)
- `description` — текстовое описание достижения

## Функциональные требования

Сервер должен реализовывать следующие эндпоинты:

- **Информация о пользователе**
- **Список всех доступных достижений**
- **Добавление новых достижений**
- **Выдача достижений пользователю** (с сохранением времени выдачи)
- **Информация о достижениях пользователя** (с переводом на выбранный язык)

### Статистические данные системы
- Пользователь с максимальным количеством достижений (в штуках)
- Пользователь с максимальной суммой баллов за достижения
- Пара пользователей с максимальной разницей в баллах
- Пара пользователей с минимальной разницей в баллах
- Пользователи, получавшие достижения 7 дней подряд (минимум одно в день)

## Опциональные задачи

- Составить `docker-compose.yml` для развертывания сервера и всех компонентов (СУБД и т.д.)
- Включить в состав Nginx в качестве reverse proxy
- Продемонстрировать навыки оформления проекта:
  - Комментарии к функциям
  - Логирование
  - Качественный README.md
- Показать умение работать с Git (история коммитов)
- Проявить фантазию в названиях и описаниях достижений
- Написать тесты для разработанного кода
```
